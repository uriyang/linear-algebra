Q1. ai 원래에 대한 개념적 이해를 위해 선형대수를 공부하는데, 실제로 ai는 벡터처럼 3차원으로 데이터를 관리해? 아니면 그보다 더 다차원적인 개념을 사용하는가?
- AI에서 데이터는 보통 “고차원 벡터”
- GPT 계열에서는 보통 수천 차원입니다.
- LM 같은 모델은 실제로 “1만 차원 이상의 공간”에서 단어 의미를 배치하고 있습니다.
- 왜 이렇게 차원이 높냐?
  > 차원이 높을수록 정보를 더 많이 담을 수 있기 때문입니다.

  - ```king - man + woman ≈ queen```같은 관계가 생기는 이유도
고차원 공간에서 의미가 구조로 표현되기 때문입니다.
- 딥러닝 모델은 "고차원 벡터 공간을 계속 회전시키고 늘리는 시스템"이다.

Q2. 각 단어 하나 당 하나의 차원을 의미하는건가?
- 예전 NLP 방식: 단어 = 차원
- 현재 AI 방식 : 단어 ≠ 차원
  - 각 차원은 단어 자체보다는 추상적인 특징(feature)를 의미함.

Q3. 추상적인 특징(feature)이라는데 embeding에서 정의한 가중치랑 영향이 있는건가? 가중지가 들어가지 않는 항목은 feature(차원)에 포함되지 않는거야?
- embedding의 각 차원(feature)은 가중치로 만들어진 결과이고, 이후 모델의 가중치들이 그 차원들을 얼마나 사용할지 결정합니다.
- Embedding 자체가 사실 “가중치 테이블”
- feature는 사람이 정의하지 않는다.
  - 모델이 학습하면서, 단어 관계를 가장 잘 표현하도록 벡터 공간을 자동으로 구성
  

Q4. 스칼라가 embeding 가중치가 될 수 있어?
- 될 수도 있고, 아닐 수도 있다
- Embedding 자체에서는 스칼라 = 가중치의 “요소”
- “선형결합의 스칼라”는 보통 다른 가중치
  - 보통 다른 레이어의 weight
  - AI에서의 스칼라의 역할
  
    | 스칼라 위치          | 의미           |
    | --------------- | ------------ |
    | embedding 내부    | feature 좌표 값 |
    | weight matrix   | feature 중요도  |
    | attention score | 벡터 섞는 비율     |

Q5. 이미 생성된 답변의 “일부 구간”에 대해서만 추가 질문(슬랙의 쓰레드)을 하면, 모델은 어떻게 연산해야 할까? 그리고 이미 끝난 연산 위에 추가 연산이 가능한가?
- 가능하다. 하지만 모델 내부에서는 “기존 계산 위에 이어서 계산”하는 게 아니라 대부분 “새로운 입력으로 다시 계산”한다.
- ChatGPT는 내부적으로 이렇게 동작합니다.
  - [지금까지 대화 전체] → 모델 → 다음 토큰 생성
- 답변이 끝났다는 의미
  - token1 → token2 → token3 → token4 ...
  - 각 단계마다 attention / 행렬곱 / 벡터연산
  - 그 토큰 생성 연산이 끝났다는 의미입니다.
- “중간 문장 질문”은 어떻게 처리될까
  - [기존 대화]
    "내가 쓴 답변"
    "사용자 질문: 2번 설명 더"
  - 실제로는 기존 연산 위에 추가 연산 이 아니라 새로운 inference
- 그럼 왜 “대화 느낌”이 나는가
  - 모델이 입력으로 받는 것이 전체 대화 history기 때문
  - 모델 입장에서는 이전 답변도 입력 텍스트
- 진짜 기술적으로 중요한 부분
  - 이미 계산된 attention을 재사용할 수 있을까?
    - LLM은 토큰을 생성할 때 이전 계산 일부를 캐싱하지만..
    - 문단 질문은 다름
