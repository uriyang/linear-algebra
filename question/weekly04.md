# PCA와 고유벡터/고유값 (AI 관점 정리)

---

## 1. 문제 상황

고차원 데이터는 다음과 같은 문제가 있다:

- 차원이 너무 큼
- 노이즈 많음
- 해석 어려움

---

## 2. 핵심 아이디어

> 데이터를 "가장 중요한 방향"으로 정리하자

---

## 3. 분산(variance)의 의미

- 분산 = 데이터가 얼마나 퍼져 있는가
- 분산이 크다 → 정보가 많다
- 분산이 작다 → 정보가 적다

---

### 직관

- 데이터가 넓게 퍼져 있는 방향 → 중요한 패턴
- 데이터가 거의 모여 있는 방향 → 의미 없음 (노이즈 가능)

---

## 4. 고유벡터 (Eigenvector)

> 데이터가 가장 많이 퍼져 있는 방향

- 방향(direction)
- 좌표축 역할
- 데이터 구조를 설명하는 축

---

## 5. 고유값 (Eigenvalue)

> 해당 방향의 "중요도"

- 크기(magnitude)
- 분산의 크기
- 정보량

---

### 핵심 관계

```text
고유값 ↑ → 중요도 ↑ → 유지
고유값 ↓ → 중요도 ↓ → 제거
```

## 6. PCA (Principal Component Analysis)
과정
1. 데이터 분포 분석
2. 고유벡터 계산
3. 고유값 기준 정렬
4. 큰 고유값 방향만 선택

결과
- 고차원 데이터 → 저차원 데이터

의미
- 핵심 구조 유지
- 노이즈 제거
- 계산 효율 증가

## 7. 선형대수 관점
𝐴𝑣 = 𝜆𝑣
- 𝑣: 고유벡터 (방향)
- 𝜆: 고유값 (중요도)
해석
- 특정 방향에서는 단순히 "크기만 변함"
- 복잡한 변환이 단순화됨

## 8. AI에서의 의미
✔ 차원 축소
1000차원 → 10차원
중요한 feature만 유지
노이즈 제거
✔ representation learning

AI는 데이터를 다음과 같이 처리한다:

입력 → 좌표변환 → 중요한 방향 선택 → 의미 생성

## 9. LLM과의 연결

PCA에서는:
- 분산 → 중요도

LLM에서는:
- attention weight → 중요도

공통점
- 중요한 정보만 선택한다

## 10. 핵심 요약
- 고유벡터 = 중요한 방향
- 고유값 = 중요도
- PCA = 중요 방향만 남기는 과정
- AI = 의미 있는 구조만 유지하는 시스템

# Q2 그러면 y = Ax에서 y의 차원이 낮아졌다면.. y와 x가 주어졌을 때 A를 구해야 하는 상황이 ai 기준으로 생길 가능성이 없어? 있다면 어떤 식으로 해결해?

> ❌ “불가능한 경우도 많다 (정보가 이미 사라졌기 때문)”
⭕ 대신 AI는 정확한 A를 찾는 게 아니라, “가장 잘 맞는 A”를 학습한다

# 차원 감소 상황에서 A를 구하는 방법 (AI 관점)

---

## 1. 최소제곱 (Least Squares)

### 문제 상황

y = Ax 이지만

- 차원 감소 발생
- 정보 손실 발생
- 정확한 해가 존재하지 않을 수 있음

---

### 해결 아이디어

정확히 맞추는 대신 오차를 최소화

---

### 수식

min ||Ax - y||^2

---

### 의미

- Ax와 y의 차이를 최소화
- 가장 가까운 해를 선택

---

### 직관

완벽한 답 없음  
→ 가장 가까운 답 선택

---

### 핵심

최소제곱은 "가장 잘 맞는 A"를 찾는 방법

---

## 2. 머신러닝 (학습)

### 구조

x → 모델 → y

---

### 내부 구조

- 모델의 파라미터 = A (가중치)
- 데이터 기반으로 학습

---

### 학습 과정

1. Ax 계산 (예측)
2. y와 비교
3. 오차 계산
4. A 업데이트

---

### 목적

오차 최소화 (loss 최소화)

---

### 의미

- 정확한 수식이 아니라
- 데이터 기반 최적화

---

### 핵심

머신러닝은 데이터를 통해 최적의 A를 학습한다

---

## 3. 정규화 (Regularization)

### 문제

- 해가 여러 개 존재
- 과적합 발생 가능

---

### 해결 아이디어

가장 단순한 해를 선택

---

### 수식

min ||Ax - y||^2 + λ||A||^2

---

### 의미

너무 복잡한 모델 X  
→ 단순한 모델 선호

---

### 효과

- 노이즈에 강함
- 일반화 성능 향상

---

### 핵심

정규화는 여러 해 중에서 가장 안정적인 해를 선택하는 방법

---

## 🔥 전체 핵심 정리

- 최소제곱 → 가장 가까운 해 찾기
- 머신러닝 → 데이터 기반 학습
- 정규화 → 가장 안정적인 해 선택

---

## 🔥 한 줄 요약

차원 감소로 인해 정확한 해를 구할 수 없는 경우,  
AI는 오차를 최소화하고 안정적인 해를 선택하는 방식으로 문제를 해결한다.

# Q3. x의 차원이 6이고, rank가 4라고 하면 ai에서 답을 얻기 위해서는 입력도4차원으로 변경하는 게 좋을까? 그리고 불필요한 입력을 찾아낼 수 있을까?
- 실제 정보가 4차원이라면, 4차원으로 압축하는 것이 더 효율적일 가능성이 큼
- 하지만 실제 정보가 4차원이라면, 4차원으로 압축하는 것이 더 효율적일 가능성이 큼
- AI에서 어떻게 하냐
- ✔ 방법 1: PCA
  ```
  데이터 → 고유벡터
    → 중요도(고유값) 기준 정렬
    → 상위 4개만 사용
  ```
  👉 자동으로 차원 축소 / 노이즈 가능
- ✔ 방법 2: 학습 기반 (딥러닝)
  ```
  모델이 알아서 중요도 학습
  ```
  👉 일부 feature weight ≈ 0 / 영향 없음
- ✔ 방법 3: feature selection
  ```
  상관관계 분석
  중요도 분석
  ```
  👉 제거 가능
> 랭크가 낮다는 건 불필요한 입력이 있다는 의미이고, 이를 줄이면 더 효율적인 모델을 만들 수 있다
